- **Breaking: `jac start --scale` defaults `app_name` to the project name, not `jaseci`** (issue #7248): with no `app_name` configured under `[scale.kubernetes]`, the deploy now names its resources after a Kubernetes-safe slug of `[project].name` (e.g. project `My App` deploys as app `my-app`) instead of the shared literal `jaseci`, so unconfigured projects no longer collide in the same namespace. An explicit `app_name` still wins, projects with no usable name keep `jaseci`, and the `namespace` default is unchanged (`default`). **Migration for existing deploys that never set `app_name`** (new users and explicitly-configured projects are unaffected): the next deploy after upgrading provisions a fresh app under the new name with empty databases, leaving the old `jaseci-*` resources -- including their data -- running alongside it. Either pin `app_name = "jaseci"` under `[scale.kubernetes]` to keep the existing identity, or adopt the new name by tearing the old deployment down first (`jac scale destroy <entry>.jac` on the pre-upgrade version).
