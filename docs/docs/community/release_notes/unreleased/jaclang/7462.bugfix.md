- **Scale: gateway pods stop installing the npm closure when the client ships prebuilt**: npm dependencies exist only to build the client bundle; when the deploy ships a host-built `.jac/client/dist`, the gateway serves it and never builds, so its bootstrap now sets `JAC_CLIENT_SKIP_NPM_INSTALL` (the `jac create --skip` knob, now also honored by `jac install`'s npm provider) and skips the closure (~10s off every gateway boot, restart, and replica scale-up). Service pods use the same knob unconditionally, replacing a jac.toml table-strip that missed the client machinery's base closure on client-kind projects. Bundles without a prebuilt dist keep the full install so the on-boot build fallback still works.
