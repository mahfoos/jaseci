- **Fix: Batch `jac check` no longer self-rejects types**: a module loaded once as a dependency and again as an explicit batch entry (e.g. via its `impl/` annex) left two declaration instances of the same class, so a type stopped comparing equal to itself and consumers checked afterwards saw spurious `X, expected X` errors that depended on which other files shared the batch. Class identity is now compared canonically (defining module, declaration site, name), so a duplicate declaration still compares equal to its twin.
