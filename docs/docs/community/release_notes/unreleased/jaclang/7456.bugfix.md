- **Fix: builtin type used as a value in client code**: a builtin type passed as a value rather than called -- `map(int, parts)`, `sorted(items, key=int)`, `f = int`, or stored in a collection -- compiled to a bare JavaScript string, so calling it threw `TypeError: "int" is not a function` in the browser while `jac check` stayed silent. Builtin types now lower to a callable runtime type object that doubles as `isinstance`/`issubclass` classinfo, and `map`/`filter` call their callback with exactly one argument as Python does.
