- **Fix: KEDA `TriggerAuthentication` names scoped per-service to prevent credential collision**: names were derived as `{trigger.name or trigger.type}-trigger-auth` with no service context, so two services sharing a trigger name (or two unnamed triggers of the same type in one spec) silently overwrote each other's credentials. Names are now produced by a shared `_trigger_auth_name` helper as `{service[:20]}-{sha256[:8]}-ta`, scoping each resource to its owning service and to the trigger's position in the spec. `apply()` also rejects any spec where two triggers resolve to the same identity, raising a clear `ValueError` before any cluster write.
