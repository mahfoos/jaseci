- **Fix: client apps showed a blank page instead of the error screen**: when a client app threw an error, the error boundary that should have displayed it crashed too, with `Uncaught ReferenceError: __jacJsx is not defined` in the console, leaving a blank page and no indication of what went wrong. The compiled client runtime referenced the JSX factory without importing it, so the error screen could never render. Errors in a client app now show the "Something went wrong" screen with the actual error message and a working "Try again" button.
