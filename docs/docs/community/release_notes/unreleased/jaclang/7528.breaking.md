- **Breaking: language clean-break -- `global`/`nonlocal` removed in favor of lexical rebinding, plus operator and keyword cleanup (issue #7514)**: A coordinated set of syntax removals with no deprecation path. (1) The `global` and `nonlocal` directive statements are gone: a bare assignment (including `+=` and `:=`) inside a function now binds to the nearest enclosing binding -- an enclosing function's local, or a module-level `glob` variable -- and creates a new local only when no such binding exists. Only `glob`-declared variables are implicitly rebindable; assigning a name that matches a module-level import, function, or class still creates a local. Shadowing an outer binding requires a typed declaration (`x: int = 5;`) placed before any use of the name in that scope (E0064 otherwise), and loop / `except ... as` / `with ... as` targets always bind fresh locals. Migration: delete `global`/`nonlocal` lines; add a typed declaration where a function intentionally shadowed a `glob` name. This also fixes the inherited Python gotcha where `x += 1` against a module global raised `UnboundLocalError`. (2) `&&` and `||` are removed -- `and`/`or` are the only spellings. (3) The abstract marker is renamed `abs` → `abst`; `abs` is again only the builtin function. (4) The parenthesized filter form `(?:Type, cond)` / `(?cond)` is removed (E0048) -- use the bracket form `[?:Type, cond]` / `[?cond]`. (5) The `root()` call form is removed (E0049) -- use bare `root` in Jac source (Python library-mode `root()` calls are unchanged). (6) Deferred fields drop the `by`: `has x: T postinit;`. (7) The C-style for loop is respelled `for i = 0 while i < 10 with i += 1 { }` -- the condition may be any full expression, and `to` is no longer a keyword (identifiers named `to` no longer need backtick escapes). (8) Backtick keyword-escaping is unified, and Python reserved words (`class`, `lambda`, ...) are now rejected as `has`-field or parameter names at compile time (E0067) instead of crashing at runtime.
