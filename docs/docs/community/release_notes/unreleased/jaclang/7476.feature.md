- **Feature: per-service `deployment_overlay` + profile-aware plugin configs**: `[scale.microservices.services.NAME.deployment_overlay]` merges a raw Deployment fragment into the generated manifest at build time (k8s-style named-list merge, identity/selector fields protected; overlays add/override only - no delete directive), so nodeSelector, probe tuning, extra volumes/env/init containers land on the FIRST rollout instead of being kubectl-patched into a second one. Plugin configs now resolve the active profile the way the CLI's env path does - `JAC_PROFILE`, the deprecated `JAC_ENV`, then `default_profile` - applying `jac.<profile>.toml` and `jac.local.toml` overlays. Previously every plugin loaded bare jac.toml, so per-environment deploy settings (e.g. a 1-replica `jac.preview.toml`) never reached the deployer. Note: the CLI's `--profile` flag is not exported to the environment, so it does not reach plugin configs yet; use JAC_PROFILE in automation.
