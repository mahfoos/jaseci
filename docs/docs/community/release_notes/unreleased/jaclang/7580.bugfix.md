- **Fix: Client entries nested below the project root**: Client entry files nested under the project root (e.g. `apps/mobile/main.jac`, or `apps/web/frontend.cl.jac` importing `shared/...`) now compile and bundle correctly. The compiled tree is rooted at the project root, so in-project dependencies stage at their project-relative paths instead of flat (which the bundler could not resolve). The React Native target computes the entry's project-relative module path (with compound extensions like `.cl.jac` stripped) instead of the bare stem, so the nested entry stages and is referenced at the correct path.
- **Fix: Service discovery for dotted `sv import` targets**: A dotted `sv import` target such as `services.ideServer` (produced when the client bridge does not sit in the service's own directory) now resolves by module stem, so the orchestrator launches the correct `services/ideServer.jac` instead of falling back to a literal `services.ideServer.jac`.
