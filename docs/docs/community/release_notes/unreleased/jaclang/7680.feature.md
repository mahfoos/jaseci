- **Feature: `@jac/mobui` gains list, overlay, and platform primitives**: The mobUI vocabulary now exports `FlatList`, `SectionList`, `RefreshControl`, `Modal`, `Switch`, `StatusBar`, `Dimensions`, `Alert`, and `Linking` alongside the existing surface. Previously mobUI authors had no virtualized list at all -- every collection had to render into a `ScrollView`, which mounts all of its children -- and no way to reach a modal, a toggle, or the alert/deep-link APIs without importing `react-native` directly and breaking out of the sealed vocabulary. All nine names resolve on both backends (React Native natively, `react-native-web` on the web target), so no new npm dependency is introduced and existing mobUI web builds are unaffected. Docs also drop the stale claim that a `Platform.select` one-liner is unavailable: `Platform` has been part of the vocabulary all along and carries `.OS` and `.select()` on both backends, so a `.native` file pair is only needed when the two platforms require different *imports*.
